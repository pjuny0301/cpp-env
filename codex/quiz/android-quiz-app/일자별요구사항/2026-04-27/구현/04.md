# Requirement 4

## 요구사항 원문

4\. 퀴즈앱으로 본문학습까지 할 수 있게(pdf 원문, 번역본(claude 생성, git에 캐싱하여 토큰절약) 비교해가며 볼 수 있음. 본문학습이 끝나면 AI랑 질답하며 한 창에서 스스로 퀴즈를 만들 수 있음)

## 구현 방식

본문학습 모드를 `content_ref` 기반으로 만든다. PDF 원문 페이지 텍스트, Claude 번역본, 이미지 ref를 나란히 보고, 학습 종료 후 같은 chapter context로 AI 질답과 즉석 퀴즈 생성을 실행한다. 번역/질답 context는 Git 캐시를 우선 조회한다.

## 관련 경로

- `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`, future worker CLI
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

앱 내부 직접 구현 대신 외부 워커/캐시 파일 중심으로 시작한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

샘플 PDF 입력, 캐시 재사용, 생성 JSON schema validation으로 검증한다.
