# Requirement 26

## 요구사항 원문

26\. 책에서 주요개념과 책에 나온 관련 설명 퀴즈앱에 MultiSelect 유형으로 그대로 매핑(인덱스로 해서 텍스트파일로 불러욤)

## 구현 방식

책의 주요개념/관련 설명은 PDF paragraph index에서 `concept_ref -> paragraph_ref -> quiz question`으로 매핑한다. MultiSelect 문제는 concept id와 정답 concept list를 저장하고, 텍스트 파일 import도 같은 schema로 변환한다.

## 관련 경로

- `apps/quiz/android-quiz-app`, `apps/quiz/quiz-vulkan`
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

React UI에서만 처리하지 말고 domain/FSM 규칙으로 먼저 고정한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

domain unit test와 주요 UI flow smoke test로 검증한다.
