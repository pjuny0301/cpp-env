# Quiz Product Big Plan

이 문서는 퀴즈 생태계 전체의 상위 폭포수 계획이다. 사용자는 이 문서를 기준으로 날짜별 요구사항을 추가하고, 구현자는 루트 `구현/NN.md`에서 하위 프로젝트로 내려가는 상위 이행 방향을 확인한다.

요구사항 번호는 실행 순서가 아니라 추적 ID다. 실제 구현은 아래 의존관계 기반 단계 순서를 따르며, 뒤 단계는 앞 단계의 계약과 검증 결과를 소비한다.

요구사항별 문서, 하위 프로젝트 계획, 현재 코드/테스트 증거는 [requirements_traceability_matrix.md](requirements_traceability_matrix.md)에서 추적한다.

## 목표

- PDF 본문/문제/이미지에서 학습 자료를 추출하고, AI로 퀴즈와 해설을 생성한다.
- 현재 Android quiz app과 quiz editor의 동작을 기준으로 삼되, 최종 제품은 native C++/Vulkan 기반 앱이 기능을 흡수해 대체한다.
- Android quiz app, quiz editor, native C++/Vulkan player가 같은 데이터 계약과 학습 규칙을 공유한다.
- AI 호출, PDF 처리, OCR, proof grading, code sandbox처럼 무거운 기능은 앱 런타임 밖의 모듈로 분리한다.
- 학습 UX는 빠르고 가볍게 유지하되, 카툰 렌더링 게임 같은 설명/대화/테마 경험까지 확장한다.

## 프로젝트 책임

- `android-quiz-app`: 현재 사용자 경험의 기준 구현이자 리메이크가 흡수할 대상이다. UX 변경은 여기서 검증한 뒤 domain/schema/FSM 규칙으로 추출한다.
- `quiz-editor`: 현재 authoring flow의 기준 구현이자 리메이크가 흡수할 대상이다. 퀴즈 데이터와 AI 산출물을 사람이 검수하고 Git에 저장하는 흐름을 표준 계약으로 만든다.
- `quiz-vulkan`: native C++/Vulkan 리메이크의 최종 수렴 지점이다. domain, scene/layout, renderer 경계를 지키며 기존 앱 기능을 C++/Vulkan 중심으로 대체한다.
- `build/external/quiz`: quiz-data, Figma export, dependencies, editor state처럼 앱 소스 밖의 외부 자료를 둔다.
- `build/out/quiz`: 설치파일, APK, target, dist, logs 같은 산출물을 둔다.
- `codex/quiz/일자별요구사항`: 날짜별 요구사항 기록, 검증 증거, 과거 plan archive를 둔다.

## 날짜별 작업 규칙

1. 새 요구는 먼저 `일자별요구사항/YYYY-MM-DD/requirements.md`에 번호와 담당 프로젝트를 적는다.
2. 요구사항 번호는 추적 ID로만 사용하고, 실제 구현 우선순위는 `의존관계 기반 실행 순서`를 따른다.
3. 요구사항별 상위 이행 방향은 루트 `구현/NN.md`에 둔다.
4. 세부 구현계획은 담당 프로젝트의 `big_plan.md`와 `구현/NN.md`에 둔다.
5. 루트 `구현/NN.md`에는 하위 프로젝트 세부 구현 순서를 중복 작성하지 않는다.
6. 과거 plan은 지우지 않고 같은 날짜의 `archive/`에 둔다.
7. plan-doc 렌더링 결과는 같은 날짜의 `generated/`에 둔다.
8. 스크린샷과 검증 자료는 같은 날짜의 `screenshots/` 또는 `verification-artifacts/`에 둔다.

2\. 퀴즈앱 퀴즈마다 디자인 업그레이드

3\. 퀴즈에디터 AI 실시간 연동(퀴즈에디터에서 AI를 부르면 퀴즈에디터 위쪽으로 실시간 연동

4\. 퀴즈앱으로 본문학습까지 할 수 있게(pdf 원문, 번역본(claude 생성, git에 캐싱하여 토큰절약) 비교해가며 볼 수 있음. 본문학습이 끝나면 AI랑 질답하며 한 창에서 스스로 퀴즈를 만들 수 있음)

5\. 위 프로젝트를 만드는데 필요한 요구사항 트리로 표현(블로그에 있는 기법과 관련시킴)

6.퀴즈를 옆으로 튕기듯 슬라이드시키면 정답표시같은 거 없이 다음 문제로 바로 넘어감.

몇번이상 문제를 슬라이드시키면 그 문제를 "이미 앎" 그룹으로 옮길 수 있는지 선택 가능.

퀴즈목록의 이미 앎 탭을 터치하면 "이미 앎" 문제를 볼 수 있고 거기서는 "이미 앎"만 있는 퀴즈를 돌릴 수 있음.

그곳 퀴즈 옵션에서는 몇번이상 틀리면 "이미 앎" 그룹에서 뺄지 입력가능

거기서 길게 터치하면 "모름"을 선턕할 수 있는 옵션이 뜸

모름을 선택하면 옵션에서 설정한 숫자와는 상관없이 "이미 앎" 그룹에서 빠지고 본 그룹으로 감

17\. **수학 수식 필기 인식 (OCR):**  11번(수학 증명) 기능과 연동하여, 태블릿에서 직접 쓴 수식을 텍스트로 변환해 AI에게 전달합니다.

18\. 수학 수식 필기 인식 (OCR): 11번(수학 증명) 기능과 연동하여, 태블릿에서 직접 쓴 수식을 텍스트로 변환해 AI에게 전달합니다.

19\. **포커스 모드 (Zettelkasten 연동):**  퀴즈를 풀다 막히면 해당 개념과 연결된 자신의 과거 메모(옵시디언 등)를 작은 팝업으로 즉시 띄워주는 기능입니다.(AI가 자신의 손필기내용을 바탕으로 생성해서 띄워줌)

20\. **망각 곡선 기반 에이징 (Spaced Repetition System, SRS):**  6번의 "이미 앎" 그룹에 들어간 문제에 시간 가중치를 부여하는 기능입니다. 며칠이 지나면 망각 곡선에 따라 해당 문제를 "이미 앎"에서 다시 "본 그룹"으로 자연스럽게 복귀시켜 장기 기억으로 전환되었는지 점검합니다.

21\. 개념 의존성 그래프 트래킹: AI가 PDF 지문을 추출할 때(12번), 단순히 텍스트만 나누는 것이 아니라 개념 간의 선후행 관계를 위상학적 트리나 방향성 비순환 그래프(DAG) 형태로 매핑합니다. 만약 14번 기능에서 AI와 토론하다가 학생이 기초 개념을 헷갈려 한다는 것이 파악되면, 즉시 그래프를 역추적하여 해당 선행 개념의 본문이나 덱을 팝업으로 추천해 줍니다.

22\. 코드/로직 스니펫 샌드박스 채점: 수학 증명(11번)과 결합하여, 프로그래밍이나 알고리즘 문제의 경우 학생이 직접 코드를 작성하면 샌드박스 환경에서 컴파일 및 실행 결과를 검증하는 유형입니다. 단순히 결과만 보는 것이 아니라, AI가 작성된 코드의 템플릿 구조나 메모리 라이프사이클 등을 분석해 피드백을 주도록 할 수 있습니다.

23\. ​리버스 엔지니어링 / 디버깅 퀴즈: AI가 의도적으로 논리적 비약이 있는 수학 증명이나, 미세한 버그(예: 참조 순환 등)가 숨겨진 개념 설명을 제시합니다. 학생이 그 '오류'를 찾아내어 지적하게 만드는 역할극 형태의 퀴즈로, 메타 인지를 강하게 자극할 수 있습니다.

24\. ​**오토마타(FSM) 기반 퀴즈 플로우 및 상태 관리:**  6번(슬라이드), 14번(AI 토론), 19번(포커스 모드) 등 유저의 액션에 따라 퀴즈 앱의 상태 전이가 굉장히 복잡해집니다. 10번(리팩토링) 단계에서 UI 플로우를 유한 상태 기계(Finite State Machine) 패턴으로 명확히 정의해두면, 애니메이션 충돌이나 예기치 않은 상태 버그를 방지하고 속도감 있는 UX를 유지할 수 있습니다

사용자가 기획하신 대규모 자동화 및 학습 시스템은 단순히 ‘앱’ 하나가 아니라, 데이터 처리-생성-학습-관리가 유기적으로 연결된 하나의 \*\*에코시스템(Ecosystem)\*\*에 가깝습니다.
​언급하신 안드로이드 앱, 데스크톱 앱, 전용 에디터 외에 전체 시스템을 완성하기 위해 필요한 구성 프로그램들을 역할별로 나열해 드립니다.
​1. 콘텐츠 생성 및 처리 엔진 (Core Engines)
​PDF 파싱 및 인덱싱 프로그램 (Requirement 12, 15): PDF에서 텍스트와 이미지를 분리하고, 장(Page)별로 인덱싱하여 파일 경로와 텍스트를 매칭하는 백엔드 프로세서입니다.
​멀티 에이전트 오케스트레이터 (Requirement 12): 메인 AI 에이전트가 지문을 분석하고, 하위 에이전트들에게 "헷갈리는 선지 생성" 등의 임무를 배분/취합하는 관리 프로그램입니다.
​수학 수식 및 필기 인식(OCR) 엔진 (Requirement 11, 17, 18): 태블릿에서 입력된 수식 이미지나 손필기를 텍스트/LaTeX로 변환하여 AI에게 전달하는 특화 엔진입니다.
​퀴즈 DSL 컴파일러/파서 (Requirement 10, 16): 전용 에디터에서 작성된 퀴즈 전용 언어를 앱에서 읽을 수 있는 표준 데이터 포맷(JSON/Protobuf 등)으로 변환해주는 파서입니다.
​2. 데이터 및 캐시 관리 시스템 (Infrastructure)
​Git 기반 번역/콘텐츠 캐시 서버 (Requirement 4): Claude가 생성한 번역본이나 퀴즈 데이터를 Git 저장소에 캐싱하여 토큰 비용을 절감하고 버전 관리를 수행하는 미들웨어입니다.
​제텔카스텐 연동 브릿지 (Requirement 19): 옵시디언(Obsidian) 등 외부 메모 앱의 로컬 폴더를 스캔하여 AI가 참조할 수 있도록 임베딩(Vectorize)하고 검색 인덱스를 만드는 프로그램입니다.
​공유 모듈 라이브러리 (Requirement 16): 안드로이드, 데스크톱, 에디터가 공통으로 사용하는 로직(이미 앎 알고리즘, 정답 검증 등)을 모아놓은 표준 라이브러리입니다.
​3. 사용자 인터페이스 (Client Apps)
​안드로이드 퀴즈 앱: 메인 학습 도구 (UX 중심).
​데스크톱 퀴즈 앱: 대화면 본문 학습 및 고도화된 수식 입력/정리를 위한 툴.
​전용 퀴즈 에디터: 퀴즈 작성 및 AI 실시간 연동(Requirement 3)이 일어나는 제작 환경.
​4. 분석 및 피드백 시스템 (Analysis & Feedback)
​수학 증명 채점 및 토론 엔진 (Requirement 11, 14): 학생의 증명 과정을 단계별로 분석하고, AI가 논리적 허점을 찾아 토론을 유도하는 대화형 엔진입니다.
​학습 통계 및 그룹 관리 시스템 (Requirement 6): "이미 앎", "모름" 그룹의 상태 전이를 추적하고 오답률에 따라 그룹에서 자동으로 제외/추가하는 로직 서버입니다.
