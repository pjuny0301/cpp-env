# 12 구현 계획

## 요구사항

12\. pdf 등록하면 AI가 넘기는 장마다 텍스트로 추출, 사진과 인덱싱
AI 에이전트가 학생들이 헷갈릴만한 지문과 개념을 하위에이전트에게 넘겨줌
\-> 그 AI는 지문내용을 바탕으로 헷걀리는 선지 섕성

## 구현 방향

PDF 등록 파이프라인은 페이지별 텍스트와 이미지 인덱스를 먼저 만들고, main agent가 헷갈릴 개념/지문 후보를 뽑아 distractor agent에 넘긴다. 하위 agent는 "그럴듯하지만 절대 정답이 아닌" 선지를 생성하고 validator가 걸러낸다.

## 핵심 산출물

- exercise classifier, sub-agent distractor generation

## 구현 순서

1. 핵심 산출물인 `exercise classifier, sub-agent distractor generation`의 입력, 출력, 저장/적용 지점을 먼저 고정한다.
2. page text, image path, paragraph id, exercise candidate를 같은 index artifact에 저장한다.
3. 헷갈리는 선지 생성은 sub-agent result로 분리해 추적 가능하게 한다.
4. worker/service command와 result artifact 형식을 정한다.
5. 에디터에는 적용 전 diff 또는 review queue로 연결한다.
6. cache miss, retry, partial failure 상태를 manifest에 남긴다.

## 대안

`exercise classifier, sub-agent distractor generation`를 외부 워커/캐시 artifact로 분리하지 않고 앱이나 에디터 내부에서 AI를 즉시 호출해 결과를 바로 반영하는 방식이 있다.

## 장단점

장점:
- 무거운 AI 작업을 앱 런타임에서 분리해 모바일/렌더러 성능을 보호한다.
- 입력, prompt version, 결과 artifact가 남아 같은 작업을 재호출하지 않고 검수할 수 있다.
- 에디터와 Vulkan/Android 앱이 같은 생성 결과를 재사용할 수 있다.

단점:
- 초기에 worker command, cache key, artifact schema를 맞추는 비용이 든다.
- 즉시 반영형 UX보다 job 상태와 review queue를 더 많이 설계해야 한다.

## 구현 기준

- 같은 입력은 같은 cache key로 재사용되어야 한다.
- AI 결과는 사람이 검수 가능한 patch나 artifact로 남아야 한다.
- 앱 런타임은 무거운 AI/OCR/분석 호출을 직접 수행하지 않아야 한다.
- 샘플 PDF 입력, 캐시 재사용, 생성 JSON schema validation으로 확인 가능해야 한다.
