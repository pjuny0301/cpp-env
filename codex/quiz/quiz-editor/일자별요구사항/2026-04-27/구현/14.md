# Requirement 14

## 요구사항 원문

14\. 정답 사유가 틀렸거나 오답이면 AI와 토론할 수 있게 한다

## 구현 방식

사유가 틀렸거나 오답이면 `discussion_session`을 연다. 토론은 원문 page/concept/question/rationale ref를 들고 시작하며, 전체 대화는 요약 캐시를 만들어 다음 호출 토큰을 줄인다.

## 관련 경로

- `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`, AI worker/cache
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

앱 내부 직접 구현 대신 외부 워커/캐시 파일 중심으로 시작한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

domain unit test와 주요 UI flow smoke test로 검증한다.
