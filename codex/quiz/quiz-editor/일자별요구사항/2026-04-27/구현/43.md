# Requirement 43

## 요구사항 원문

43\. 정답만 채점 / 과정도 채점 옵션에서 선택가능

## 구현 방식

정답만 채점/과정도 채점은 `grading_mode` 옵션으로 둔다. 정답만 모드는 빠르고 싸게, 과정 모드는 느리지만 오류 위치와 reasoning feedback을 제공하게 비용을 명확히 표시한다.

## 관련 경로

- `build/external/tools`, `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

한 번에 전체 구현하지 말고 schema, state, UI, test 순서로 나눈다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

고정 샘플 입력과 expected grader/OCR output fixture로 검증한다.
