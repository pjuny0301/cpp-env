# Requirement 44

## 요구사항 원문

44\. 정답 채점시 태블릿에서 어느 부분이 왜 틀렸는지 AI가 직접 스크롤해서 그 부분을 가리키면서 설명

## 구현 방식

태블릿에서 어느 부분이 틀렸는지 가리키는 기능은 feedback anchor로 구현한다. AI가 bbox/stroke range를 반환하면 UI가 해당 위치로 스크롤하고 highlight/pointer overlay를 표시한다.

## 관련 경로

- `build/external/tools`, `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

앱 내부 직접 구현 대신 외부 워커/캐시 파일 중심으로 시작한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

고정 샘플 입력과 expected grader/OCR output fixture로 검증한다.
