# Requirement 70

## 요구사항 원문

70\. pdf가 본문 + 문제 형식인 경우에는 문제는 퀴즈로 보내고 본문은 앱에다 배치함. 특정 본문을 설명하기 위해 가장 잘 맞는 형식으로 이미지 여러장 혹은 한 장 생성(예시: 학습만화 등)

## 구현 방식

PDF가 본문+문제 형식이면 classifier가 본문 block과 exercise block을 분리한다. 본문은 reader content로, 문제는 quiz JSON으로 보내며, 설명용 이미지는 chapter artifact로 생성해서 캐시한다.

## 관련 경로

- `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`, future worker CLI
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

앱 내부 직접 구현 대신 외부 워커/캐시 파일 중심으로 시작한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

샘플 PDF 입력, 캐시 재사용, 생성 JSON schema validation으로 검증한다.
