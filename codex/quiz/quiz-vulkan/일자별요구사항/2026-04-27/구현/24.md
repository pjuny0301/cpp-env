# Requirement 24

## 요구사항 원문

24\. ​**오토마타(FSM) 기반 퀴즈 플로우 및 상태 관리:**  6번(슬라이드), 14번(AI 토론), 19번(포커스 모드) 등 유저의 액션에 따라 퀴즈 앱의 상태 전이가 굉장히 복잡해집니다. 10번(리팩토링) 단계에서 UI 플로우를 유한 상태 기계(Finite State Machine) 패턴으로 명확히 정의해두면, 애니메이션 충돌이나 예기치 않은 상태 버그를 방지하고 속도감 있는 UX를 유지할 수 있습니다

사용자가 기획하신 대규모 자동화 및 학습 시스템은 단순히 ‘앱’ 하나가 아니라, 데이터 처리-생성-학습-관리가 유기적으로 연결된 하나의 \*\*에코시스템(Ecosystem)\*\*에 가깝습니다.
​언급하신 안드로이드 앱, 데스크톱 앱, 전용 에디터 외에 전체 시스템을 완성하기 위해 필요한 구성 프로그램들을 역할별로 나열해 드립니다.
​1. 콘텐츠 생성 및 처리 엔진 (Core Engines)
​PDF 파싱 및 인덱싱 프로그램 (Requirement 12, 15): PDF에서 텍스트와 이미지를 분리하고, 장(Page)별로 인덱싱하여 파일 경로와 텍스트를 매칭하는 백엔드 프로세서입니다.
​멀티 에이전트 오케스트레이터 (Requirement 12): 메인 AI 에이전트가 지문을 분석하고, 하위 에이전트들에게 "헷갈리는 선지 생성" 등의 임무를 배분/취합하는 관리 프로그램입니다.
​수학 수식 및 필기 인식(OCR) 엔진 (Requirement 11, 17, 18): 태블릿에서 입력된 수식 이미지나 손필기를 텍스트/LaTeX로 변환하여 AI에게 전달하는 특화 엔진입니다.
​퀴즈 DSL 컴파일러/파서 (Requirement 10, 16): 전용 에디터에서 작성된 퀴즈 전용 언어를 앱에서 읽을 수 있는 표준 데이터 포맷(JSON/Protobuf 등)으로 변환해주는 파서입니다.
​2. 데이터 및 캐시 관리 시스템 (Infrastructure)
​Git 기반 번역/콘텐츠 캐시 서버 (Requirement 4): Claude가 생성한 번역본이나 퀴즈 데이터를 Git 저장소에 캐싱하여 토큰 비용을 절감하고 버전 관리를 수행하는 미들웨어입니다.
​제텔카스텐 연동 브릿지 (Requirement 19): 옵시디언(Obsidian) 등 외부 메모 앱의 로컬 폴더를 스캔하여 AI가 참조할 수 있도록 임베딩(Vectorize)하고 검색 인덱스를 만드는 프로그램입니다.
​공유 모듈 라이브러리 (Requirement 16): 안드로이드, 데스크톱, 에디터가 공통으로 사용하는 로직(이미 앎 알고리즘, 정답 검증 등)을 모아놓은 표준 라이브러리입니다.
​3. 사용자 인터페이스 (Client Apps)
​안드로이드 퀴즈 앱: 메인 학습 도구 (UX 중심).
​데스크톱 퀴즈 앱: 대화면 본문 학습 및 고도화된 수식 입력/정리를 위한 툴.
​전용 퀴즈 에디터: 퀴즈 작성 및 AI 실시간 연동(Requirement 3)이 일어나는 제작 환경.
​4. 분석 및 피드백 시스템 (Analysis & Feedback)
​수학 증명 채점 및 토론 엔진 (Requirement 11, 14): 학생의 증명 과정을 단계별로 분석하고, AI가 논리적 허점을 찾아 토론을 유도하는 대화형 엔진입니다.
​학습 통계 및 그룹 관리 시스템 (Requirement 6): "이미 앎", "모름" 그룹의 상태 전이를 추적하고 오답률에 따라 그룹에서 자동으로 제외/추가하는 로직 서버입니다.
​요약: 전체 아키텍처 구성도 예시
​Ingestion: PDF 인덱싱 프로그램
​Generation: AI 에이전트 오케스트레이터 (번역/선지 생성)
​Storage: Git 캐시 서버 + 제텔카스텐 브릿지
​Authoring: 전용 언어 에디터 (AI 실시간 연동)
​Learning: 모바일/데스크톱 앱 (UX, 슬라이드 인터랙션, 수학 OCR)
​Refinement: 증명 채점 및 토론 엔진
​이렇게 보니 단순한 앱 개발을 넘어 \*\*"AI 기반 개인용 지식 베이스(Personal Knowledge Base) 구축 엔진"\*\*을 만드시는 셈이네요. 이 중에서 가장 먼저 '모듈화'하여 독립시키고 싶은 핵심 프로그램은 무엇인가요?

계정·권한 관리 백엔드: 로그인/회원관리, 학습 데이터 저장, 기기 간 동기화를 담당하는 서버입니다.
API 서버 및 공통 서비스 모듈: 각 클라이언트(앱, 에디터 등)와 AI 엔진 사이를 연결하는 REST/GraphQL API를 제공합니다. 번역·질문 생성·채점 요청을 처리하고, 캐싱·토큰 관리 등을 수행합니다.
데이터 분석·대시보드 시스템: 학습 성과를 시각화하고 사용자별 강점/약점을 분석하여 리포트를 제공하는 모듈입니다.
보안·프라이버시 모듈: 민감한 학습 데이터를 암호화하고 AI 요청 로그를 익명화하며, GDPR/국내 규정에 맞춰 데이터 처리를 검증합니다.
애플리케이션 업데이트·배포 자동화: 안드로이드/데스크톱 앱과 에디터의 버전 관리, 자동 빌드 및 배포를 돕는 CI/CD 파이프라인입니다.
품질 관리 및 검수 도구: AI가 생성한 번역·문항·채점 결과를 미리 점검하는 QA용 내부 도구로, 오류·편향을 줄입니다.
사용자 접근성 기능: 텍스트‑투‑스피치, 화면크기/색상 설정 등 다양한 사용자를 위한 보조 기능을 패키징한 라이브러리입니다.

각각의 기능을 독립된 프로그램으로 분리하고, 외부 개발자가 쉽게 참여할 수 있도록 표준을 정의하려면 모듈화 설계가 중요합니다. 일반적인 모듈화·플러그인 아키텍처의 원칙에 따르면, 복잡한 애플리케이션을 “기능별로 나눠진 작은 모듈”로 설계해야 하며, 각 모듈은 자체 기능과 리소스를 소유하고 다른 부분과 느슨하게 결합되어야 합니다�. 또한 모듈 간 인터페이스를 명확히 정의하면 내부 구현을 숨기면서도 독립적으로 배포·테스트할 수 있어 개발 효율을 높일 수 있습니다�.

## 구현 방식
