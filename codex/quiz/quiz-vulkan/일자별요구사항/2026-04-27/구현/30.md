# Requirement 30

## 요구사항 원문

30\. 왼쪽으로 슬라이드하면 이전 퀴즈 볼 수 있게

## 구현 방식

왼쪽 스와이프 이전 문제는 history stack과 current index rewind로 구현한다. 이미 제출된 feedback을 되살릴지, 단순 복습으로 볼지는 FSM 정책으로 분리하고 setting으로 선택 가능하게 한다.

- `codex/quiz/big_plan.md`

## 대안

React UI에서만 처리하지 말고 domain/FSM 규칙으로 먼저 고정한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

domain unit test와 주요 UI flow smoke test로 검증한다.
