# Requirement 64

## 요구사항 원문

64\. Deck은 마치 첨부할 암기고래 앱같이 3 x n걔 박스가 늘어선 형태로 하고 Day는 마찬가지로 암기고래같이 얇은 게 가로로 늘어서게

## 구현 방식

Deck 화면은 3 x N 박스 grid, Day는 얇은 horizontal strip으로 디자인한다. `apps/quiz/quiz-vulkan`에서는 responsive grid/strip layout token으로 만들고, 모바일/데스크톱별 column count를 layout test로 고정한다.

## 관련 경로

- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

React UI에서만 처리하지 말고 domain/FSM 규칙으로 먼저 고정한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

domain unit test와 주요 UI flow smoke test로 검증한다.
