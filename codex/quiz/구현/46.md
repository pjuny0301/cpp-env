# 46 상위 구현 연결

## 요구사항

46\. pdf 텍스트 불러올시 오른쪽에 흐릿한 번역본 창 띄움. 움직이지 않으며 테두리가 없고 옆에 있는 본문이랑 같은 크기이며 텍스트에디터 창 안에 있음. 연보라 배경에 그보단 약간 어두우면서도 세련된 보라색 글씨

## 상위 이행 방향

이 루트 문서는 요구사항 46를 세부 구현하지 않고 `android-quiz-app`, `quiz-vulkan`의 waterfall 작업으로 내려보내는 상위 연결 문서다. 실제 구현 방식, 코드 경계, 검증 기준은 하위 프로젝트 `big_plan.md`와 `구현/46.md`에서 관리한다.

## 담당 프로젝트

- `android-quiz-app`: [android-quiz-app/big_plan.md](../android-quiz-app/big_plan.md), [android-quiz-app/구현/46.md](../android-quiz-app/구현/46.md) - native C++/Vulkan 리메이크가 흡수할 사용자 경험 기준과 모바일 동작 검증을 담당한다.
- `quiz-vulkan`: [quiz-vulkan/big_plan.md](../quiz-vulkan/big_plan.md), [quiz-vulkan/구현/46.md](../quiz-vulkan/구현/46.md) - 공통 domain, scene/layout, renderer, 장기 C++/Vulkan 포팅을 담당한다.

## 폭포수 이행 순서

1. 루트 `big_plan.md`에서 요구사항 원문을 확인한다.
2. 이 문서에서 담당 프로젝트와 하위 문서 진입점을 확인한다.
3. `android-quiz-app/big_plan.md`에서 프로젝트 내 단계와 선행조건을 확인한다.
4. `android-quiz-app/구현/46.md`에서 실제 구현 방향, 대안, 장단점, 구현 기준을 따른다.
5. `quiz-vulkan/big_plan.md`에서 프로젝트 내 단계와 선행조건을 확인한다.
6. `quiz-vulkan/구현/46.md`에서 실제 구현 방향, 대안, 장단점, 구현 기준을 따른다.
7. 프로젝트 간 공유 데이터나 상태 의미가 있으면 루트 문서에는 계약 이름만 남기고, 실제 구현 세부사항은 각 프로젝트 문서에서 관리한다.

## 대안

루트 `quiz/구현` 문서에 모든 세부 구현계획을 쓰고, 하위 프로젝트 `big_plan.md`와 `구현/`에는 같은 내용을 복사하는 방식이 있다.

## 장단점

장점:
- 상위 문서는 요구사항과 하위 진입점만 관리하므로 프로젝트 세부 구현과 겹치지 않는다.
- 구현자는 `big_plan -> quiz/구현 -> project big_plan -> project/구현` 순서로 읽으면 된다.
- 프로젝트별 세부 구현 변경이 루트 문서와 충돌할 가능성이 줄어든다.

단점:
- 한 요구사항을 완전히 이해하려면 루트 문서와 담당 프로젝트 문서를 함께 열어야 한다.
- 담당 프로젝트가 바뀌면 루트 연결 문서와 프로젝트 인덱스를 같이 갱신해야 한다.

## 이행 기준

- 루트 문서에는 하위 프로젝트 세부 구현 순서나 코드 경계를 중복 작성하지 않는다.
- 모든 담당 프로젝트의 `big_plan.md`와 `구현/NN.md` 링크가 실제 파일로 열려야 한다.
- 하위 구현이 바뀌면 해당 프로젝트 문서를 먼저 고치고, 루트 문서는 담당/흐름 변경이 있을 때만 갱신한다.
