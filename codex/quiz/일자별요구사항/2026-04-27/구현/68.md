# Requirement 68

## 요구사항 원문

68\. 풀면서 문제 보이면 폰에서 즉시 수정할 수 있게

## 구현 방식

폰에서 즉시 수정은 lightweight editor mode로 구현한다. 문제 화면에서 edit action을 열고, 변경은 local patch로 저장한 뒤 Git sync queue에 넣어 충돌 시 에디터에서 해결하게 한다.

## 관련 경로

- `apps/quiz/quiz-editor`, `apps/quiz/quiz-vulkan`, future worker CLI
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

React UI에서만 처리하지 말고 domain/FSM 규칙으로 먼저 고정한다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

domain unit test와 주요 UI flow smoke test로 검증한다.
