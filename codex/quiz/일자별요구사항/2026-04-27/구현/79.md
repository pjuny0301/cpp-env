# Requirement 79

## 요구사항 원문

79\. 위 계획들을 일관적으로 시행하기 위해서는 어찌해야하는지?

## 구현 방식

일관 실행을 위해 ADR, schema version, owner map, wave plan, test matrix, token budget을 고정한다. 새 요구는 반드시 schema/domain/UI/render/service 중 어디에 속하는지 분류한 뒤 구현한다.

## 관련 경로

- `codex/quiz`, `apps/quiz/quiz-vulkan/docs`
- `codex/quiz/big_plan.md`
- `codex/quiz/big_plan.md`

## 대안

한 번에 전체 구현하지 말고 schema, state, UI, test 순서로 나눈다.

## 장단점

장점: 포팅과 검증이 쉬워진다. 단점: 초기에 문서와 계약을 먼저 맞추는 비용이 든다.

## 검증

domain unit test와 주요 UI flow smoke test로 검증한다.
